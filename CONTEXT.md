# Daily Tech Briefing Site Context

This file is the project vocabulary and operating map for maintainers and AI agents. Read it before changing scheduling, launchd, cache, feedback, qmd, or health-check behavior.

## Product

Daily Tech Briefing Site is a local-first publishing layer for scheduled technology intelligence. It does not collect news by itself. It reads Markdown reports generated by an upstream collector, builds a web cache, serves a readable public site, accepts reader feedback, and writes feedback and maintenance records back to Markdown.

## Source Of Truth

- Markdown reports in `NEWS_ARCHIVE_DIR` are the content source of truth.
- Feedback Markdown in `FEEDBACK_DIR` is the reader-feedback source of truth.
- Maintenance Markdown in `MAINTENANCE_DIR` is the human-readable operations trail.
- Cache files are derived runtime state and may be rebuilt.
- launchd templates in `launchd/templates` are source templates; installed plists are rendered runtime artifacts.

## Modules

- `server.js`: HTTP API, static site, feedback intake, local maintenance API.
- `src/config.js`: environment parsing, schedule contract, cache and directory paths.
- `src/report-parser.js`: Markdown report parsing.
- `src/site-index.js`: cache build/read logic.
- `src/feedback-store.js`: feedback Markdown writer.
- `src/ops-store.js`: maintenance Markdown and status writer.
- `scripts/check-refresh.js`: delayed web-publish follow-up for collection slots.
- `scripts/install-launchd.sh`: renders launchd templates, support wrappers, runtime copy, and optional qmd/tunnel agents.
- `scripts/run-qmd-refresh.sh`: optional qmd collection refresh wrapper.
- `scripts/send-feedback-health-report.js`: reference health and feedback report aggregator.

## Operating Contracts

- Daily collection slots must be configurable with `DAILY_COLLECTION_SLOTS`.
- Reference production defaults are morning-only.
- Web refresh follows collection after a configurable lag and retries for late reports.
- Afternoon and evening refresh agents must remain opt-in, not installed by default.
- Optional qmd refresh is installed only when `WIKI_SOURCE_DIR` is configured.
- launchd services must run through support wrappers, not directly against mutable development state.
- Public package checks must pass before publishing: `npm run check`, `npm run smoke`, and `npm run audit:schedule`.

## Commercial Readiness Bar

A change is commercially acceptable only when it preserves:

- reproducible install,
- explicit configuration,
- clear source/runtime separation,
- privacy-safe public artifacts,
- meaningful smoke and schedule tests,
- human-readable incident and operations records,
- no hidden dependency on the original operator's machine.
