# Daily Tech Briefing Site

Current version: `1.1.7`

Chinese version: [README.zh-CN.md](README.zh-CN.md)

Daily Tech Briefing Site is a local-first publishing layer for scheduled technology intelligence. It turns AI, ICT, and semiconductor briefing Markdown generated by an OpenClaw collection pipeline into a readable website, collects reader feedback, and optionally pushes refresh alerts or feedback digests through a bound OpenClaw channel.

The public package contains no private tokens, no personal paths, no bundled runtime state, and no private source archive. Users provide their own report directory, feedback directory, maintenance token, channel target, tunnel token, and OpenClaw setup.

## 2026-06-03 Update

- The recommended tunnel path now prefers a user-level `http2` connector pinned to fixed Cloudflare edge IPs for networks where `argotunnel` SRV discovery is unstable.
- The `10:15` feedback and health receipt now distinguishes the current installed OpenClaw version from the latest automated unified-upgrade record.
- The feedback-health wrapper can recover `FEISHU_TARGET` from the active OpenClaw cron contract before treating the push target as missing.
- The collector-side Juya YouTube fallback now tolerates a stale `youtube-feed` response by continuing to `yt-dlp` when the source is required for the morning slot.

## 2026-06-04 Operations Contract

- The reference production contract is morning-only: `09:40` collection and `10:15` feedback/health receipt.
- Collection slots, collection time, web refresh lag, retry count, and retry interval are configurable through `.env`.
- Afternoon and evening collection/refresh jobs are disabled by default and are not installed unless explicitly opted in through `DAILY_COLLECTION_SLOTS` or installer flags.
- Morning refresh defaults to tracking late reports with 36 attempts at 10-minute intervals, starting 20 minutes after the reference collection time.
- launchd live services read the support cache under `~/Library/Application Support/daily-tech-site/cache`; project `.cache` is for local/manual runs.

## 2026-06-05 Reliability Contract

- qmd refresh is optional, explicitly enabled by `WIKI_SOURCE_DIR`, and removed from launchd when not configured.
- launchd installation reads private support `site.env` when present, so local runtime paths do not have to enter the public package.
- qmd refresh LaunchAgent declares a working directory and runs through the support wrapper.
- Health receipts distinguish cron structural failures from recent execution warnings.
- The package now includes a 1+N maintenance guide, commercial readiness review, and incident log.

## Purpose

The goal is to make daily technology monitoring reusable, inspectable, and easier to improve as a community:

- Collect AI, ICT, cloud, data center, semiconductor, edge AI, and enterprise AI engineering signals from layered sources.
- Publish daily snapshots as a stable web reading experience.
- Push important updates through configured channels.
- Keep feedback, maintenance logs, refresh checks, privacy checks, and schedule gates auditable.
- Let operators replace source pools, add new sources, tune schedules, customize push frequency, and improve the pipeline together.

Core flow:

```text
OpenClaw scheduled collector
  -> AI / ICT / semiconductor Markdown reports
  -> this website cache and APIs
  -> public web reading page
  -> reader feedback Markdown
  -> optional feedback digest
  -> optional OpenClaw channel push
  -> local health and privacy gates
```

## What This Package Does

- Reads scheduled Markdown briefing files from `NEWS_ARCHIVE_DIR`.
- Builds a cache for a clean web reading experience.
- Serves a public briefing site and a local-only maintenance page.
- Saves reader feedback as Markdown.
- Summarizes feedback with a configured model wrapper, or falls back to deterministic rule-based suggestions.
- Optionally sends refresh alerts or feedback digests through OpenClaw Feishu broadcast when `FEISHU_TARGET` is configured.
- Provides launchd templates for macOS operators.
- Provides privacy, syntax, plist, schedule, and smoke gates before publishing.

## Quick Start

```bash
cp .env.example .env
npm run build:cache
npm run dev
```

Open:

```text
http://localhost:4321
```

The repository includes one sample report in `data/collections`, so the site can run immediately after clone even before you connect your own OpenClaw collector.

## Configuration

Edit `.env` before production use:

```bash
PORT=4321
HOST=0.0.0.0
SITE_TITLE=Daily Tech Briefing
FIXED_SITE_URL=http://localhost:4321
NEWS_ARCHIVE_DIR=./data/collections
FEEDBACK_DIR=./data/feedback
FEEDBACK_DIGEST_DIR=./data/feedback/_digest
MAINTENANCE_DIR=./data/maintenance
CACHE_DIR=./.cache
DAILY_COLLECTION_SLOTS=morning
MORNING_COLLECTION_TIME=09:40
MORNING_REFRESH_LAG_MINUTES=20
MORNING_REFRESH_MAX_ATTEMPTS=36
MORNING_REFRESH_RETRY_DELAY_MINUTES=10
SUMMARIZE_WRAPPER=
OPENCLAW_BIN=openclaw
FEISHU_ACCOUNT=default
FEISHU_TARGET=
MAINTENANCE_TOKEN=replace-with-a-local-maintenance-token
```

Required for production:

- `NEWS_ARCHIVE_DIR`: directory containing generated Markdown reports.
- `FEEDBACK_DIR`: directory where reader feedback Markdown files are written.
- `MAINTENANCE_DIR`: directory where runtime logs and status notes are written.
- `MAINTENANCE_TOKEN`: token required for the maintenance page and local maintenance APIs.

Optional integrations:

- `SUMMARIZE_WRAPPER`: a shell wrapper that accepts prompt text on stdin and returns feedback suggestions on stdout. In the reference OpenClaw setup this points to `summarize-pro` through `summarize-openclaw.sh`.
- `OPENCLAW_BIN`, `FEISHU_ACCOUNT`, `FEISHU_TARGET`: channel push through OpenClaw Feishu broadcast. Other OpenClaw channels can be wired by adapting `src/feishu.js`.
- `.env.tunnel`: Cloudflare Tunnel token for `npm run run:tunnel`.

## Source Collection

The reference upstream collector is OpenClaw `daily_news_v10.py`. Its default product scope is AI, ICT, and semiconductor intelligence. The source scope can be replaced after clone by changing source manifests, keywords, schedules, and OpenClaw collector configuration.

The reference collection flow is layered:

1. News sites, official blogs, and official website mirrors.
2. WeChat official-account mirrors, discovery search, and direct seed URLs.
3. Video and podcast creator pools.
4. AI Builder pool powered by Follow Builders.
5. Relevance, date, dedupe, and ranking gates.
6. `summarize-pro` for selected final items only.
7. Markdown output consumed by this website.

The collector dependency chain includes Scrapling, urllib fallback, Steel.dev or browser tooling where needed, WeChat readers, yt-dlp or video probe tooling, Follow Builders, `summarize-pro`, Kimi or fallback model wrappers, Cloudflare Tunnel, qmd refresh jobs, and OpenClaw channel push.

The website package does not bundle private source archives or private credentials. New public sources are welcome through the Source Request issue template.

## Model Chain

In the reference setup, `summarize-pro` is used as the summarization wrapper for selected final items and feedback digests. It may call Kimi depending on your OpenClaw environment.

If you do not use Kimi, keep the wrapper contract the same:

- Accept prompt text on stdin.
- Return a concise Markdown result on stdout.
- Handle failures without breaking the website.
- Keep deterministic fallback behavior available for smoke tests and local operation.

## Channel Push

Channel push is optional. When `FEISHU_TARGET` is configured, the site can send refresh alerts or feedback digests through OpenClaw Feishu broadcast. Operators can adapt `src/feishu.js` to other OpenClaw-supported channels.

Daily push time and frequency are configurable through launchd templates, cron, or your own scheduler. The public package includes schedule tests so changes remain inspectable.

## macOS launchd

The public repo commits templates under `launchd/templates`. Running:

```bash
npm run install:launchd
```

generates real plists into `~/Library/LaunchAgents` using your current project path and log directory. It does not require or contain any private path from the original machine.

Before installing launchd:

1. Create `.env`.
2. Set `MAINTENANCE_TOKEN`.
3. Set `FEISHU_TARGET` if you want push alerts.
4. Create `.env.tunnel` only if you want Cloudflare Tunnel.
5. Set `WIKI_SOURCE_DIR` only if you want the optional qmd refresh job.

## Privacy Boundary

The repository intentionally ignores:

- `.env`, `.env.*`, except examples.
- `.env.tunnel`.
- `.cache/`.
- `data/feedback/`.
- `data/maintenance/`.
- logs, dependencies, and build outputs.

Run before publishing:

```bash
npm run audit:privacy
```
